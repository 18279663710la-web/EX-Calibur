
项目名称： CloudRAG-Hub：基于云原生架构的多模态智能文件检索与知识库管理系统
项目方向： 人工智能方向 - RAG 智能体（文件知识库检索）
核心技术栈： Svelte (前端) + Go/Python (BFF代理后端) + Dify (AI编排) + Qdrant (向量数据库) + Docker (容器化部署)

一、 项目目标与预期成果
本项目旨在开发一套移动端优先的智能文件检索与知识库管理系统。通过融合 RAG（检索增强生成）技术与云原生架构，解决传统文件检索中存在的语义理解不足和信息孤岛问题。
预期成果为一个可通过公网访问的完整 Web 应用程序。用户可跨平台（尤其是移动端）无缝上传各类格式文档，并通过自然语言与智能体进行流式对话，实现对文档内容的精准检索、总结与提炼，甚至辅助后续的内容创作。

二、 系统架构与实现方案
系统采用经典的三层解耦架构，结合本地后端代理模式与云端大模型算力，确保系统的安全性、高性能与可扩展性。

1. 交互模块（LUI/GUI - 移动端优先）：

技术框架： 采用先进的 Svelte (TypeScript) 框架配合 Tailwind CSS 开发。

实现逻辑： 严格遵循响应式设计原则，确保侧边栏导航和聊天界面的布局在手机屏幕下自动适配，提供接近原生 App 的丝滑体验。前端通过 SSE (Server-Sent Events) 技术接收后端的流式文本，实现打字机效果。

2. 核心后端与代理中间件（BFF 架构）：

技术框架： 采用 Go (Gin/Echo) 或 Python (FastAPI) 构建自研后端服务。

实现逻辑： 不在前端直接暴露大模型的 API Key。所有前端请求（如文件上传、对话查询）先进入自研后端，由后端进行鉴权、日志记录后，再在服务器内部向 Dify API 发起代理请求。此架构彻底消除了安全隐患，并为后续的数据埋点提供支持。

3. 人工智能与知识库引擎：

模型编排： 依托 Dify 平台构建文件知识库与 RAG 智能体工作流。

算法原理： 采用混合检索（Hybrid Search）方案。将传统的 BM25 关键词检索与基于大模型 Embedding 接口的语义向量检索进行 RRF（倒数秩融合）加权，最后引入 Rerank 重排模型，大幅提升复杂文档查询的准确率。大语言模型推理（如 DeepSeek 或 GLM）均通过云端 API 调用，极大释放本地算力。

4. 基础数据与大数据处理：

数据存储： 采用 PostgreSQL 存储用户基础信息与业务状态；采用 Qdrant 等专业向量数据库持久化存储文档分块后的向量数据。

数据分析： 后端服务实时采集用户操作日志、接口响应延迟（Latency）及模型 Token 消耗量，并在数据库中沉淀。前端规划管理员仪表盘，对系统运行成本与用户设备来源进行可视化统计，体现大数据应用价值。

三、 云原生环境部署
为保证项目交付的工程化标准，整体系统采用虚拟化与容器化技术进行统一部署。

实现方式： 编写 docker-compose.yml 脚本，将前端服务、代理后端、PostgreSQL 数据库进行容器编排。

网络配置： 部署于云服务器（如阿里云/腾讯云），配置反向代理与公网 IP，确保答辩现场可通过互联网直接访问，支持多用户并发体验。

四、 核心业务时序流转
文件上传： 客户端上传文件 -> 自研后端接收并记录数据库 -> 转发至 Dify -> Dify 调用大模型提取文本并向量化 -> 存储至向量数据库 -> 返回成功状态。

知识检索： 客户端发起自然语言查询 -> 自研后端拦截并统计请求头 -> 代理请求 Dify 智能体 -> 触发 RAG 检索并连接云端 LLM 推理 -> 结果以流式 (Stream) 返回后端 -> 后端实时透传至客户端。
